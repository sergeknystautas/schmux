# Design Specs

Design documents for features that are **not yet fully implemented**. Once a feature is complete, its spec is consolidated into a subsystem guide in `docs/` using the `/finalize` command, and the spec file is deleted.

## Lifecycle

1. **Brainstorm** → design spec written here (`docs/specs/`)
2. **Plan** → implementation plan written in `docs/plans/`
3. **Implement** → code written, plan deleted
4. **Finalize** → spec consolidated into `docs/*.md` guide, spec deleted

## Current specs

Files in this directory describe features that are in progress or planned but not yet built. Do not treat these as accurate descriptions of the current codebase — they describe intended designs that may not match what was actually implemented.
